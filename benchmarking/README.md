# Benchmarking

This directory contains the fixed sample benchmark used to compare the video pipeline against external model samples.

## What Is Included

The default artifact root is:

```bash
benchmarking/artifacts/fixed30_qwen36
```

It contains:

- `manifest.json`: selected prompts, model rows, and video paths.
- `videos/`: evaluated videos for each model.
- `scores/results.json`: per-sample Qwen 3.6 Flash judge results.
- `results.md`: generated summary table.

The fixed sample contains 30 prompts:

- 15 from `Complex_Plot`
- 15 from `Multi-View_Consistency`

The currently tracked model set is:

- `our`: videos generated by this repository's Prefect pipeline.
- `veo3`: sampled VBench2 videos from the Hugging Face dataset.
- `wanx`: sampled VBench2 videos from the Hugging Face dataset.
- `vidu_q1`: sampled VBench2 videos from the Hugging Face dataset.

## Scoring

`run_sample_bench.py` evaluates videos with OpenRouter video review using:

```text
qwen/qwen3.6-flash
```

For each video it stores:

- `overall_score`
- `prompt_adherence`
- criterion-level scores and rationales
- transport metadata for the video upload path

This is a VBench2-like API proxy benchmark. It is useful for regression tracking and model comparison on the fixed sample, but it is not the official GPU VBench2 score.

## Environment Configuration

Benchmark scripts use the same runtime configuration as the agent. Configure
secrets in:

```text
video_scene_agent/.env
```

Create the file from the package template:

```bash
cp video_scene_agent/.env.example video_scene_agent/.env
```

For judging existing benchmark videos, set at least:

```bash
OPENROUTER_API_KEY=...
STORAGE_PATH=./artifacts
```

For regenerating this pipeline's benchmark videos, also set:

```bash
KLING_ACCESS_KEY=...
KLING_SECRET_KEY=...
```

The benchmark directory does not have its own `.env`. Keep local secrets only in
`video_scene_agent/.env`; it is ignored by git.

## Commands

Print or refresh the summary table from existing saved scores:

```bash
python3 benchmarking/run_sample_bench.py report
```

Judge missing or forced samples with Qwen 3.6 Flash:

```bash
python3 benchmarking/run_sample_bench.py judge --model all --allow-paid
```

Judge one model:

```bash
python3 benchmarking/run_sample_bench.py judge --model our --allow-paid
```

Judge one sample:

```bash
python3 benchmarking/run_sample_bench.py judge --model our --sample-id complex_plot_000 --allow-paid
```

Regenerate this pipeline's videos for selected samples through the Prefect flow:

```bash
python3 benchmarking/generate_vbench2_samples.py --sample-id complex_plot_000 --allow-paid
```

Generate all samples through the Prefect flow:

```bash
python3 benchmarking/generate_vbench2_samples.py --allow-paid
```

Live judging and generation read secrets from `video_scene_agent/.env`. Do not commit that file.

## Failure Recovery

The benchmark writes results incrementally after each judged sample. If judging fails midway, rerun the same command without `--force`; completed rows are skipped.

Generation also copies each completed video into the artifact root immediately and updates `manifest.json` after each sample, so a later rerun can continue from the existing videos.
